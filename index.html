Smart Plant Marketplace Platform with AI Vision & Chatbot

01. Problem

"กลุ่มผู้ซื้อต้นไม้ประสบปัญหาต้นไม้หลังจากซื้อไปไม่นานเนื่องจากขาดความรู้ในการดูแลและไม่สามารถวินิจฉัยโรคพืชได้อย่างถูกต้อง นอกจากนี้ ผู้ซื้อยังขาดความมั่นใจในการซื้อต้นไม้ออนไลน์ ขณะที่ผู้ขายมีข้อจำกัดในการเข้าถึงลูกค้าโดยตรง ผู้ใช้งานยังต้องการช่องทางในการเรียนรู้เกี่ยวกับการปลูกต้นไม้ การขอคำแนะนำเมื่อเกิดปัญหา และการเข้าร่วมกิจกรรมหรือเวิร์กชอปเพื่อพัฒนาทักษะการดูแลต้นไม้"

จึงเกิดแนวคิดในการออกแบบแพลตฟอร์มที่รวมการซื้อขายต้นไม้ การให้คำแนะนำผ่าน AI การวิเคราะห์โรคพืชจากรูปภาพ และระบบจองเวิร์กชอปไว้ในที่เดียว เพื่อช่วยให้ผู้ใช้งานเข้าถึงข้อมูลได้ง่ายขึ้นและมีความมั่นใจตลอดการใช้งาน

70คน

กลุ่มผู้ซื้อ เจอปัญหาเกี่ยวกับต้นหลังจากซื้อไป

20คน

ประสบความยากลำบากในการจองเวิร์กชอป

02. User Persona

เราทำการสัมภาษณ์เชิงลึกกับผู้ซื้อ และเจ้าของร้านขายต้นไม้ ณ อุทยานเกษตร มหาวิทยาลัยขอนแก่น เพื่อสร้าง Persona ที่สะท้อนปัญหาจริงในตลาด

คุณกิ่ง, 28 (User)

บุคลากรงานในมข.

Goals & Needs
  • ต้องการปลูกต้นไม้ฟอกอากาศเพื่อสร้างบรรยากาศที่ผ่อนคลายภายในที่พัก แต่ไม่มีความรู้ด้านการดูแลต้นไม้
  • ต้องการคำแนะนำการดูแลต้นไม้ที่เหมาะกับพืชแต่ละชนิด
  • อต้องการจองเวิร์กชอปออนไลน์ได้สะดวก พร้อมดูรายละเอียดและรอบเวลาได้ด้วยตนเอง
Pain Points
  • ค้นหาข้อมูลการดูแลต้นไม้จากหลายแหล่ง แต่ข้อมูลไม่ชัดเจนและนำไปใช้จริงได้ยาก
  • ไม่มั่นใจว่าต้นไม้ที่กำลังมีอาการผิดปกติเกิดจากโรคพืชหรือการดูแลที่ไม่ถูกต้อง
  • ต้องติดต่อร้านโดยตรงเพื่อสอบถามข้อมูลหรือจองเวิร์กชอป ทำให้เสียเวลาและไม่สะดวก

คุณเมฆ, 35 (Store Seller)

ร้านค้า ณ อุทยานเกษตร มข.

Goals & Needs
  • ต้องการระบบจัดการสต็อกสินค้าพืชและโปรโมชั่นด้วยตนเอง
  • ต้องการเปิดคอร์สจองสอนเวิร์กชอปทำสวนขวดสร้างรายได้เสริม
  • ลดเวลาตอบแชทคำถามทั่วไปเพื่อเน้นขยายยอดขาย
Pain Points
  • ต้องพึ่งพาพ่อค้าคนกลางและแบกรับต้นทุนการตลาดที่สูงขึ้น
  • ไม่มีเครื่องมือจัดสรรระบบจองคิวเวิร์กชอปที่เป็นระเบียบ
  • ต้องรับโทรศัพท์จากลูกค้าที่โทรเข้ามาสอบถามเพื่อลงทะเบียนจองทำเวิร์กชอป
03. Customer Journey Map

การวิเคราะห์ Journey ตลอดวงจรชีวิตการปลูกพืชของผู้ใช้งาน เพื่อแก้ไข Pain Points ของการซื้อต้นไม้ทั่วไป

Stage 1: Discovery

ค้นหาข้อมูล

ผู้ใช้เสิร์ชหาภาพพืชฟอกอากาศแต่ไม่รู้สายพันธุ์จริง ทำให้สั่งซื้อผิดพลาด

UX Solution (Realized)

ระบบ AI วิเคราะห์รูปภาพต้นไม้ จำแนกชนิดพืชพร้อมรายละเอียดประกอบการตัดสินใจทันที

Stage 2: Purchase

เลือกสั่งซื้อสินค้า

ผู้ซื้อกลัวพืชเสียหายระหว่างขนส่ง หรือข้อมูลผู้ขายไม่น่าเชื่อถือ

UX Solution (Realized)

หน้าดีเทลยืนยันสุขภาพ มีระบบตะกร้า ตรวจสอบความถูกต้องของสลิป ชำระเงินปลายทาง หรือ QR Code และการเคลมพืชป่วย

Stage 3: Offline Sync

จองคิวเวิร์กชอป

ต้องการฝึกฝนทักษะการจัดตู้/สวนขวดด้วยวิธีที่ถูกต้องแบบกลุ่มและเจอผู้สอนจริง

UX Solution (Realized)

ระบบจองเวิร์กชอปทำสวนขวด แบบลงทะเบียนในปฏิทิน เลือกหัวข้อ และรอบเวลาเรียนระบุค่ามัดจำ

Stage 4: Post-Care

การดูแลในระยะยาว

พืชแสดงอาการเหี่ยวเฉา แผ่นใบเป็นคราบขาวจุดดำ ไม่แน่ใจวิธีการแก้ไขปัญหา

UX Solution (Realized)

ระบบ Chatbot อัจฉริยะ(OPENAI GPT API Integration) บนเว็บไซต์คอยให้ข้อมูลดูแลแบบรายบุคคลตลอด 24 ชม.

04. System Architecture

จากการออกแบบโครงสร้างในโปรเจกต์ ระบบได้ประยุกต์ร่วมกับแพลตฟอร์มของ Line Developer, Webhook API ผ่าน Make.com และเรียกประมวลผลโมเดลภาษา OpenAI เพื่อส่งกลับข้อมูลโรคและคู่มือพืชไปยังผู้ใช้ได้เรียลไทม์

ส่วนติดต่อผู้ใช้ (Frontend)

HTML5, CSS3, JavaScript, และจัดความสอดคล้องอินเทอร์เฟซด้วย Bootstrap & CSS Framework

บริการฝั่งเซิร์ฟเวอร์ (Backend)

Laravel Framework ในการจัดการ Controller, Routing, Authentication, และการโต้ตอบ API

ระบบข้อมูล & อัลกอริทึม

ฐานข้อมูล MySQL, OpenAI GPT Core API, และระบบ Line OA webhook สั่งประมวลคำตอบแบบพลวัต

05. Data Modeling

การออกแบบสถาปัตยกรรมข้อมูลที่มีเสถียรภาพและรองรับการขยายตัว (Scalability) ด้วยการทำนอร์มัลไลเซชัน (Normalization) ออกมาเป็นโครงสร้าง ER-Diagram รวมทั้งหมดกว่า 26 ตาราง ตัวอย่างตารางหลักจากระบบ:

Table Name Key Fields & PK/FK UX/System Description
Users user_id (PK), name, Email, postcode, bookingworkshop_id (FK) เก็บข้อมูลพฤติกรรม พื้นฐานการจัดส่ง และลิงก์ประวัติการจองคอร์ส
Sellers sellers_id (PK), user_id (FK), store_name, owners_name, address ระบบร้านค้าย่อยของอุทยานเกษตรเพื่อทำตลาดแยกสต็อกพืชของตนเอง
Products product_id (PK), sellers_id (FK), category_id (FK), price, stock_quantity เก็บข้อมูลพืช ขนาด รหัสผู้ขาย จำนวนในสต็อก และยอดขายเพื่อวิเคราะห์
Claims claims_id (PK), order_id (FK), user_id (FK), type (Refund/Claim), status ระบบประกันสุขภาพพืชระหว่างจัดส่ง ผู้ซื้อสามารถแนบภาพ/วิดีโอยื่นขอเคลมได้
Workshop_bookings workshop_bookings_id (PK), user_id (FK), booking_code, quantity, payment_status จัดการตั๋วคิวเพื่อความจุห้องเรียนสูงสุดตามกฎข้อจำกัดของร้านค้า (Sessions)

07. Design Decisions
1

ลดความสับสนด้านสายพันธุ์ของต้นไม้

ผู้ซื้อต้นไม้ออนไลน์มีแนวโน้มตัดสินใจด้วยความรู้สึกทางสายตาและการรับรู้ชื่อที่เป็นที่รู้จัก เราออกแบบ UI หน้าผลลัพธ์การค้นหาเพื่อผสานงานถ่ายภาพพืชที่จัดแต่งสวยงาม แสดงผลร่วมกับรายละเอียดความต้องการปัจจัยทางธรรมชาติที่สัญลักษณ์สากลอ่านง่าย (น้ำ, แสง, ขนาด) เพื่อให้ผู้เริ่มต้นเลือกพืชที่เหมาะกับคอนโดได้ทันที

2

แก้ปัญหาวงจรสั่งทำสวนขวดแบบกลุ่มผ่านระบบจองเวลา

จากที่ผู้ใช้เคยหงุดหงิดกับการจองสวนขวดผ่านแชทแล้วไม่มีแอดมินตอบเป็นเวลานาน เราออกแบบระบบ Workshop Session Scheduling ให้ระบุวันที่และเวลารอบเรียนแบบ Dynamic Slot แสดงความจุคิวที่เหลือชัดเจน และกำหนดขั้นตอนมัดจำก่อนยืนยันคิวผ่านหน้าประวัติการจอง ช่วยลดอัตราการหลุดคิวเรียนลงมากกว่า 80%
